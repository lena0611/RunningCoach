# PaceLAB 메뉴 정보 구조

작성일: 2026-05-29 · 최종 갱신: 2026-07-03 (리디자인 ①b·①c — 5탭 + 계정 드로어 반영)

## 목적
요약, 코치, 기록, 추세, 기억 메뉴의 성격을 러너 관점에서 분리해 각 화면에 들어갈 콘텐츠와 제외할 콘텐츠를 정한다.

기준 도메인 흐름은 `목표 -> 러닝 기록 -> 부하와 적응/몸 상태 -> 코칭 -> 다음 훈련`이다.

## 상위 원칙
- 하단 메뉴는 기능 묶음이 아니라 러너의 질문 단위로 나눈다.
- `요약`은 지금 무엇을 해야 하는지 답한다. 북극성: 앱을 열면 "나 오늘 뭐하지?"가 1초 안에 해소된다.
- `코치`는 왜 이 훈련인지, 계획이 무엇인지 답한다. 그룹핑 원칙: 기억=코칭 엔진의 입력, 코치=산출.
- `기록`은 내가 무엇을 했는지 확인하고 고친다.
- `추세`는 내가 어떻게 변하고 있고 무엇을 조정해야 하는지 답한다.
- `기억`은 앱과 코치가 나를 어떤 기준으로 알고 있는지 관리한다.
- 세션별 AI 코칭 "대화"는 특정 세션 맥락이 필요하므로 세션 상세에서 열며, App 레벨 오버레이로 어느 탭 위에서든 뜬다.
- Upload는 독립 메뉴가 아니다. 기록 추가 액션이며 `기록` 안의 보조 액션으로 둔다.
- 프로필·업적·설정은 탭이 아니라 **우상단 계정 드로어(AppHeader)** 소관이다(①c 확정 — 기억 탭에서 제거).
- 같은 데이터가 여러 메뉴에 보일 수는 있지만, 편집 권한과 주 해석 위치는 하나만 둔다.

## 메뉴별 역할 (5탭, 2026-07 리디자인 확정)
| 메뉴 | 러너의 질문 | 주 역할 | 시간 축 |
| --- | --- | --- | --- |
| 요약 | 오늘 무엇을 해야 하지? | 오늘의 처방(TodayHero) 중심 즉답 | 현재 |
| 코치 | 왜 이 훈련이지? 계획은? | 세션 브리핑, 주간 계획, 페이즈, 레벨/퀘스트, 코치 모먼트 | 현재~가까운 미래 |
| 기록 | 내가 무엇을 어떻게 뛰었지? | 세션 원장, 상세 분석, 세션별 코칭 진입 | 과거 세션 |
| 추세 | 나는 어떻게 변하고 있지? | 종합 판단 + 렌즈 drill-in, 다음 처방 영향 | 누적 |
| 기억 | 앱이 나를 어떤 러너로 알고 있지? | 코칭 기준 요약 + 목표/몸 상태/훈련 기준/AI 기억 관리 | 장기 맥락 |

## 요약
### 역할
요약은 러너의 현재 작전판이다. 주인공은 TodayHero(오늘의 처방) 하나 — 무엇·얼마나·페이스 + 시작 버튼. 나머지는 그 답을 받치는 조연.

### 포함 콘텐츠 (①b 확정 레이아웃)
1. WeekStrip: 월~일 요일 칩, 오늘 강조, 타입 dot, 완료 표시. 탭하면 코치 탭.
2. TodayHero: 오늘의 처방(세션 타입 삽화 배경) + CTA. 휴식 중에는 복귀 컨트롤 히어로.
3. NumbersGrid(2×2): 주간 거리, Easy 비율, 강훈련, 평균 심박.
4. CoachInsight 1개.
5. RecentRuns 2~3행.
6. 필요한 경우 부상 프로브·휴식 배너·복귀 램프(안전·상태 신호는 요약 홈 잔류 — 감사 결정).

### 제외 콘텐츠
- 레벨/퀘스트/페이즈/세션 브리핑/리스케줄 → 코치 탭으로 이동(①b). 레벨은 헤더 칩으로만.
- 전체 기록 목록과 월별 캘린더, 세션 상세 차트, 긴 편집 폼, import 전용 화면.

## 코치 (①b 신설)
### 역할
요약에서 분리한 계획·코칭·동기 화면. "왜 이 훈련인지 / 계획은?"에 답한다.

### 포함 콘텐츠
- L1: SessionBriefingCard · WeekTrainingCarousel · TrainingPhaseCard · LevelCard · QuestPanel · CoachMomentCard.
- L2(StackPage push): 세션 브리핑 상세 · 레이스 준비(RacePage) · AI 코칭 대화.
- L2+(바텀시트): 리스케줄 · 휴식 선언 · 더블 추가 · 주말 트리아지.

## 기록
### 역할
기록은 러닝 세션 원장이다. 저장된 `RunLog`를 찾고, 추가하고, 수정하고, 삭제하고, 세션별 상세와 코칭을 확인한다. 밀도는 유지한다(리디자인 감사에서 "좋은 모델"로 평가).

### 포함 콘텐츠
- 전체 러닝 목록, 날짜별/월별 그룹, 캘린더(타입색 마커, 다중 세션 카운트 배지).
- 필터: BottomSheetSelect ×2(세션 유형·메타 태그).
- 월 요약: 달리기 수·시간·킬로칼로리·거리·페이스, 전체/평균 2열.
- 기록 추가: HealthKit 동기화 상태, FIT import, 수동 추가.
- 기록 수정/삭제, 세션 상세 진입(App 레벨 오버레이), 세션별 AI 코칭, HealthKit 새로고침.

### 제외 콘텐츠
- 활성 목표 전체 편집, 장기 코칭 기억 편집, 주간 루틴 전체 편집, 부상 항목 전체 관리, 앱 전체 설정.

## 추세
### 역할
장기 변화를 한 판단으로 요약한다(①c에서 밀도 낮춤). 첫인상은 종합 판단 1개.

### 포함 콘텐츠
- 비교 기준 선택(기간 vs 기간 프리셋).
- 종합 판단 hero: 톤 배지(좋은 흐름/주의) + 한 줄 결론 + 신뢰도.
- 렌즈 리스트 5개(목표까지·같은 심박에서·무리했나·잘 수행했나·회복됐나): 좌측 액센트 바 = 판단색, 값 + 판단 + chevron.
- 렌즈 상세(StackPage drill-in): 그 지표 hero, 요약 카드, 시각화, 해석, 다음 처방 영향, 근거 세션(세션 상세 오버레이로 연결).

### 제외 콘텐츠
- 5개 지표의 L1 동시 펼침, 원본 기록 수정, 장기 기억 편집.

## 기억
### 역할
기억은 코치의 장기 맥락 관리 화면이다(①c에서 축소). "나를 어떤 기준으로 코칭할 것인가"만 관리한다. 대시보드도, 기록 목록도, 계정 관리 화면도 아니다.

### 포함 콘텐츠 (①c 확정 레이아웃)
- L1: **현재 코칭 기준 요약 카드**(활성 목표 + 제약 2행) + **관리 nav 4항목**(목표 · 몸 상태(주의 배지) · 훈련 기준 · AI 기억) + "프로필·업적은 우상단 계정 메뉴" 안내.
- L2/L3(drill-in): 목표 목록/편집/신규 · 부상 목록/편집/신규(부위 선택기 포함) · 훈련 기준(훈련 단계, 장거리 전략, 볼륨 노트, 주간 루틴) · AI 기억(승급 조건, 처방 템플릿, 장기 메모 4종 편집) · 훈련 지식 보관소/검토 요청.
- 저장은 **항목별 저장**: 열려 있는 패널 그룹만 저장한다. 전역 저장 버튼은 없다.

### 제외 콘텐츠
- 러너 프로필·PB·업적(→ 계정 드로어), 최근 세션 목록 반복 표시, 전체 기록 캘린더, 세션 상세 차트, 일시적 컨디션 요약, raw AI 대화 전문.

## 계정 드로어 (탭 아님 — AppHeader 우상단)
- 계정 정보(루트) → 개인정보 수정(프로필: 출생연도/성별/경력/체중/러너 레벨/주간 횟수/롱런 요일/심박 기준/PB) · 설정(알림) · **업적**(①c에서 기억 탭으로부터 이전) · 용어 안내.
- 드로어는 우→좌 스택, 내부 2차 패널은 back 화살표. 러너 프로필·PB·업적의 편집·열람 원본 위치는 여기다.

## 콘텐츠 배치 기준
| 콘텐츠 | 주 위치 | 보조 노출 | 이유 |
| --- | --- | --- | --- |
| 활성 목표 | 기억 | 요약, 코치 | 목표의 원본 관리는 장기 맥락이고, 요약/코치는 현재 판단 기준만 보여준다. |
| 다음 훈련 추천 | 요약 | 코치(주간 캐러셀) | 지금 할 일을 결정하는 콘텐츠다. |
| 세션 브리핑·페이즈·레벨·퀘스트 | 코치 | 헤더 레벨 칩 | 계획·동기 산출물은 코치 소관이다(①b). |
| 전체 RunLog | 기록 | 요약 최근 세션 | 기록은 과거 세션 원장이다. |
| 세션 상세 | App 레벨 오버레이 | 요약/기록/추세에서 진입 | 어느 탭에서 열든 같은 상세를 그 탭 위에 띄운다. |
| 세션별 AI 코칭 | 세션 상세 CTA | 코치 탭 | 코칭 대화는 선택 세션 맥락 기준이다. |
| AI 장기 기억 | 기억 | 없음 | 장기 기억은 코칭 기준이며 원장이나 작전판이 아니다. |
| 부상 체크인 | 전역/요약 CTA | 기억 관리 | 체크인은 놓치면 안 되는 행동이고, 항목 관리는 기억이다. |
| 부상 항목 편집 | 기억 | 요약 위험 신호 | 장기 제한 조건이다. |
| 날씨 | 요약 | 기록의 세션 저장값 | 다음 훈련 준비와 과거 세션 해석에만 쓴다. |
| 기록 추가 | 기록 | 없음 | 저장 원장에 새 항목을 추가하는 액션이다. |
| 사용자 프로필/PB | **계정 드로어** | 요약 목표 준비도 근거 | 개인화 기준이되 코칭 "기억" 관리와는 분리(①c). |
| 업적 | **계정 드로어** | 달성 순간 celebrate 토스트 | 회고·수집 콘텐츠는 탭 정보 구조 밖이다(①c). |
| 주간 루틴 | 기억(훈련 기준) | 요약 다음 세션 | 루틴 원본은 장기 기준이고, 오늘 실행할 부분만 요약한다. |

## UI 후속 기준
- 기억 페이지 1차 구조는 `현재 코칭 기준 요약 → 관리 nav(목표/몸 상태/훈련 기준/AI 기억) → drill-in 편집(항목별 저장)`이다. 한 화면에 편집 폼을 나열하지 않는다.
- 요약 페이지는 TodayHero 중심을 유지하고, 기억 편집성 콘텐츠나 코치 산출물을 끌어오지 않는다.
- 기록 페이지는 원장/상세/코칭 진입을 유지하되, 기록 추가 액션을 명확히 한다.
- 구현 세부 CSS, 컴포넌트 배치, 카드 밀도, 모바일 인터랙션은 `ui-guidelines.md`와 `ui-system-contract.md`에서 결정한다.
