# PaceLAB 메뉴 정보 구조

작성일: 2026-05-29

## 목적
요약, 기록, 기억 메뉴의 성격을 러너 관점에서 분리해 각 화면에 들어갈 콘텐츠와 제외할 콘텐츠를 정한다.

기준 도메인 흐름은 `목표 -> 러닝 기록 -> 부하와 적응/몸 상태 -> 코칭 -> 다음 훈련`이다.

## 상위 원칙
- 하단 메뉴는 기능 묶음이 아니라 러너의 질문 단위로 나눈다.
- `요약`은 지금 무엇을 해야 하는지 답한다.
- `기록`은 내가 무엇을 했는지 확인하고 고친다.
- `기억`은 앱과 코치가 나를 어떤 기준으로 알고 있는지 관리한다.
- AI Coach는 독립 메뉴가 아니다. 특정 세션 코칭은 `기록`의 세션 상세에서 열고, 누적 코칭 기준과 장기 기억은 `기억`에서 관리한다.
- Upload는 독립 메뉴가 아니다. 기록 추가 액션이며 `기록` 안의 보조 액션으로 둔다.
- 같은 데이터가 여러 메뉴에 보일 수는 있지만, 편집 권한과 주 해석 위치는 하나만 둔다.

## 메뉴별 역할
| 메뉴 | 러너의 질문 | 주 역할 | 시간 축 |
| --- | --- | --- | --- |
| 요약 | 오늘/이번 주 무엇을 해야 하지? | 현재 상태 판단과 다음 행동 제안 | 현재와 가까운 미래 |
| 기록 | 내가 무엇을 어떻게 뛰었지? | 세션 원장, 상세 분석, 세션별 코칭 | 과거 세션 |
| 기억 | 앱이 나를 어떤 러너로 알고 있지? | 장기 목표, 몸 상태, 코칭 기준 관리 | 장기 맥락 |

## 요약
### 역할
요약은 러너의 현재 작전판이다. 앱을 열었을 때 가장 먼저 다음 훈련, 목표 흐름, 회복/부상 위험, 최근 훈련 신호를 짧게 판단한다.

### 포함 콘텐츠
- 활성 목표 요약: 목표 거리, 목표 기록, 목표일, 남은 기간.
- 다음 훈련 추천: 날짜/요일, 세션 타입, 핵심 처방, 조절 사유.
- 오늘 또는 다음 세션 준비: 날씨, 더위/비/바람, 회복/통증 체크 포인트.
- 목표 예상 준비도: 수행능력, 역치/템포, 유산소 베이스, 롱런 기반, 일관성, 부상/회복 게이트.
- 최근 7일/30일 핵심 흐름과 이번 달 누적 거리: 거리, 횟수, 강도 분포, 급격한 증가 경고.
- 최근 세션 1~5개: 상세 진입을 위한 요약 카드.
- 필요한 경우 부상 체크인 또는 코칭 기준 확인 CTA.

### 화면 구조 (2026-06-02 확정)
화면 순서는 우선순위와 동일한 1열 흐름이며 모바일/데스크톱 공통이다.

1. 다음 훈련 Hero: 오늘/다음 예정 구분, 세션 타입, 주간 루틴/누적 helper, 추천일 날씨 한 줄. 탭하면 핵심 처방, 조절 사유, 날씨 상세(WeatherCard)를 담은 다음 훈련 상세 스택을 연다. 다음 세션 날씨는 별도 카드가 아니라 이 Hero와 상세 스택 안에서만 보여준다.
2. 활성 목표 카드: 목표 제목, 목표 거리/기록, 목표일/D-day, 예상 기록과 준비도. 탭하면 목표 예상 상세 스택, 예상 산출 불가 시 기억의 목표 패널로 이동한다.
3. 몸 상태 신호: 부상 기준 카드(탭 시 기억 부상 패널)와 피로 경고 카드(안정/주의 + 근거, 탭 시 최근 7일 추이 스택).
4. 최근 훈련 흐름: 이번 달, 최근 7일, Easy 비율(30일), 강훈련(7일) 메트릭. 각 카드는 추이 스택으로 진입한다.
5. 최근 세션 최대 5개: 세션 상세 스택 진입, 전체 보기는 기록 탭으로 이동한다.

### 제외 콘텐츠
- 전체 기록 목록과 월별 캘린더.
- 세션 상세 차트, 랩 전체 목록, 경로 상세.
- 목표/프로필/부상 항목의 긴 편집 폼.
- 모든 AI 장기 기억 목록.
- 데이터 import 전용 화면.

### 우선순위
1. 다음 훈련 추천.
2. 활성 목표와 목표 준비도.
3. 회복/부상/날씨 경고.
4. 최근 훈련 흐름.
5. 최근 세션 진입.

## 기록
### 역할
기록은 러닝 세션 원장이다. 저장된 `RunLog`를 찾고, 추가하고, 수정하고, 삭제하고, 세션별 상세와 코칭을 확인한다.

### 포함 콘텐츠
- 전체 러닝 목록, 날짜별/월별 그룹, 캘린더.
- 기록 추가: HealthKit 동기화 상태, FIT import, 수동 추가.
- 기록 수정/삭제.
- 세션 상세: 거리, 시간, 페이스, 심박, 케이던스, RPE, 코스 타입, 메타 태그.
- 랩/스플릿, 페이스/심박/케이던스/고도 차트, 경로 상세.
- 세션별 AI 코칭 스레드와 추가 질문.
- 세션별 HealthKit 새로고침.
- 기록 필터: 타입, 코스, 입력 소스, 날씨/스플릿/차트/경로 데이터 보유, 사용자 태그.

### 제외 콘텐츠
- 활성 목표 전체 편집.
- 장기 코칭 기억 편집.
- 주간 루틴 전체 편집.
- 부상 항목 전체 관리.
- 앱 전체 설정.

### 우선순위
1. 최근/선택 날짜의 기록 접근.
2. 세션 상세와 세션별 코칭.
3. 기록 추가/수정/삭제.
4. 필터와 개인 시상/분석.
5. HealthKit/FIT 데이터 보강.

## 기억
### 역할
기억은 코치의 장기 맥락 관리 화면이다. 러너가 앱에게 알려준 목표, 프로필, 몸 상태, 훈련 기준, 코칭 기억을 확인하고 수정한다.

기억은 대시보드가 아니고, 기록 목록도 아니다. "나를 어떤 기준으로 코칭할 것인가"를 관리하는 곳이다.

### 포함 콘텐츠
- 사용자/선택 러너: 이름, 선택 사용자, 기본 프로필.
- 목표: 활성 목표, 보조 목표, 목표 전략, 목표일, 성공 기준.
- 러너 프로필: 나이, 성별, 러닝 경력, 주간 목표 횟수, 선호 롱런 요일, PB.
- 몸 상태/부상: active/monitoring injury, 정규화 부위, painLevel, 제한 조건, 복귀 기준, 보강운동 기준.
- 훈련 기준: AI가 관리하는 weeklyPattern, 장거리 전략, 현재 볼륨 노트, heat strategy, running style.
- 적응형 코칭 프로필: 훈련 단계, 승급/유지/하향 조건, 처방 템플릿, 반복 준수 패턴.
- 코칭 장기 기억: AI가 유지하는 성향, 반복 패턴, 장기 메모.
- 훈련 지식 요청/승인 상태가 제품 범위에 들어오면 이 화면의 하위 섹션으로 둔다.

### 제외 콘텐츠
- 최근 세션 목록의 반복 표시.
- 전체 기록 캘린더.
- 세션 상세 차트.
- 오늘 다음 훈련 카드의 전체 복제.
- 매번 바뀌는 날씨/일시적 컨디션 요약.
- raw AI 대화 전문 전체 목록.

### 우선순위
1. 활성 목표와 보조 목표.
2. active 부상/주의사항과 회복 제한.
3. 러너 프로필과 PB.
4. 현재 주간 루틴과 훈련 기준.
5. AI 장기 기억과 적응형 프로필.
6. 확장 후보인 훈련 지식 요청.

## 콘텐츠 배치 기준
| 콘텐츠 | 주 위치 | 보조 노출 | 이유 |
| --- | --- | --- | --- |
| 활성 목표 | 기억 | 요약 | 목표의 원본 관리는 장기 맥락이고, 요약은 현재 판단 기준만 보여준다. |
| 다음 훈련 추천 | 요약 | 없음 | 지금 할 일을 결정하는 콘텐츠다. |
| 전체 RunLog | 기록 | 요약 최근 세션 | 기록은 과거 세션 원장이다. |
| 세션 상세 | 기록 | 요약 최근 세션에서 진입 | 세션별 차트와 코칭은 특정 기록 맥락이 필요하다. |
| 세션별 AI 코칭 | 기록 | 없음 | 코칭은 선택 세션 날짜와 수행 맥락을 기준으로 한다. |
| AI 장기 기억 | 기억 | 요약에서 기준 일부 표시 | 장기 기억은 코칭 기준이며 원장이나 작전판이 아니다. |
| 부상 체크인 | 전역/요약 CTA | 기억 관리 | 체크인은 놓치면 안 되는 행동이고, 항목 관리는 기억이다. |
| 부상 항목 편집 | 기억 | 요약 위험 신호 | 장기 제한 조건이다. |
| 날씨 | 요약 | 기록의 세션 저장값 | 다음 훈련 준비와 과거 세션 해석에만 쓴다. |
| 기록 추가 | 기록 | 없음 | 저장 원장에 새 항목을 추가하는 액션이다. |
| 사용자 프로필/PB | 기억 | 요약 목표 준비도 근거 | 개인화 기준이다. |
| 주간 루틴 | 기억 | 요약 다음 세션 | 루틴 원본은 장기 기준이고, 오늘 실행할 부분만 요약한다. |

## UI 후속 기준
- `03-ui-ux`는 기억 페이지를 한 화면에 모든 카드를 나열하는 구조가 아니라 장기 맥락 섹션 구조로 재설계한다.
- 기억 페이지의 1차 섹션은 `목표`, `몸 상태`, `러너 프로필`, `훈련 기준`, `AI 기억` 순서를 우선 검토한다.
- 요약 페이지는 다음 훈련 추천과 목표 준비도 중심으로 유지하고, 기억 편집성 콘텐츠를 끌어오지 않는다.
- 기록 페이지는 원장/상세/코칭 진입을 유지하되, 기록 추가 액션을 명확히 한다.
- 구현 세부 CSS, 컴포넌트 배치, 카드 밀도, 모바일 인터랙션은 `03-ui-ux`에서 결정한다.
